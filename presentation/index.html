Terraform

About terraform

Why not Heat?

Is it ready for prime time ?

We needed to able to build/destroy our environments at will

chose to use terraform as it is easier to GROK than heat

We /could/ have done this with heat.

Perfect for dev/test stuff don't care about. Probably neither it or you are mature enough for prod.

DEIS

Docker based PAAS

think "Heroku" behind the firewall

Focus on UX

v1 on CoreOS+Fleet, v2 on K8s

I've worked with DEIS for quite some time and it's UX and Herokuesque workflow made it a no brainer.

Heavy focus on stateless applications using external services (trove,RDS,etc) for data persistence.

It is Docker based, currently runs on top of CoreOS+Fleet

Moving to run on top of k8s in next major version.

Relatively easy and quick to install and use, very easy to automate with Terraform.

Docker Swarm

Simple multi-server docker container orchestration

TLS/certs for security / authorization

For running things that don't run well in PAAS.

Why not K8s ?

We knew some stuff wouldn't run well in PAAS due to state

Swarm is easy to set up and run using Terraform

Uses TLS for auth / encryption

UX between docker and swarm is identical, K8s has steeper learning curve

k8s too complicated for our simple needs here.

Docker Registry

Running on Swarm

Backed by swift

custom jenkins and other images saved here.

We run registry on each swarm node, only listens on localhost

Backed by swift so our images persists and is available to all.

We host our customized jenkins image and some other stuff in there.

Jenkins

Used official jenkins image, added deis binaries and keys and stuff, saved off with docker commit.
